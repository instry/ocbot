## Phase 2: Claude-Compatible Skills + Evolution

**Goal**: Align skill format with Claude SKILL.md spec. Add triggerPhrases for fast matching. Improve self-heal quality via structured SKILL.md.

### 2.1 triggerPhrases Matching (HIGH PRIORITY)

**Files**: `lib/skills/types.ts`, `lib/skills/matcher.ts`, `lib/skills/create.ts`

**Problem**: Current `matchSkill()` uses URL hostname + name keyword matching, which is imprecise. Semantic LLM matching is available but disabled (too slow for every message).

**Solution**: Add `triggerPhrases: string[]` to Skill type. Generate during skill creation via LLM. Match via fast text inclusion check.

**Steps**:
1. Add `triggerPhrases: string[]` to `Skill` interface
2. Update `createSkillFromExecution` LLM prompt to generate 3-5 trigger phrases
3. Update `matchSkill()`:
   - Phase 1: `skill.triggerPhrases.some(p => normalizedMessage.includes(p))`
   - Phase 2 (existing): URL hostname + name keywords
   - Phase 3 (optional): LLM semantic match
4. Update `SkillEditPage` to allow editing trigger phrases
5. Backfill: for existing skills without triggerPhrases, extract from name + description

**Verification**: Create a skill, verify triggerPhrases are generated. Test matching with various phrasings.

### 2.2 Structured SKILL.md Generation (HIGH PRIORITY)

**Files**: `lib/skills/create.ts`

**Problem**: Current `skillMd` is free-form markdown generated by LLM. Quality varies, and L3 heal doesn't have reliable structure to work with.

**Solution**: Update LLM prompt to generate SKILL.md with standardized sections matching Claude's format.

**Template for LLM prompt**:
```
Generate a SKILL.md document with this structure:

---
name: <kebab-case-name>
description: <one-sentence description>
triggerPhrases:
  - "<phrase 1>"
  - "<phrase 2>"
  - "<phrase 3>"
startUrl: "<url>"
categories:
  - <category>
parameters:
  - name: <param_name>
    type: string|number|boolean|select
    description: "<description>"
    required: true|false
---

# <Skill Title>

## Workflow
1. <step 1>
2. <step 2>
...

## Preconditions
- <precondition 1>

## Success Criteria
- <criterion 1>

## Notes
- <edge case or limitation>
```

**Steps**:
1. Update `createSkillFromExecution` to use structured prompt
2. Parse frontmatter from generated SKILL.md to populate Skill fields
3. Store full SKILL.md (frontmatter + body) in `skillMd` field
4. Update `createAutoSkill` to generate minimal structured skillMd

**Verification**: Create skills from various task types, verify SKILL.md has all sections.

### 2.5 Skill Dry-Run / Eval Framework (MEDIUM PRIORITY)

**Problem**: No way to test a skill without running it for real. Editing a skill requires trial-and-error.

**Solution**: Add "Test Run" capability:
1. Run skill in current tab with step-by-step visual feedback
2. Mark as test execution (doesn't affect score)
3. Show pass/fail per step with screenshots
4. For fragile steps: pre-check xpath resolvability without executing

**Steps**:
1. Add `dryRun: boolean` option to `SkillRunner.execute()`
2. In dry-run mode: capture screenshot after each step, report step-by-step results
3. Add "Test" button on SkillEditPage
4. Show test results inline (step list with green/red indicators)

## Phase 4: Cloud + Marketplace

**Goal**: Skill sharing, discovery, and ecosystem.

### 4.1 Cloud Skill Store

- Backend API: CRUD + sync for private skills
- Client sync: upload encrypted blobs, incremental pull by timestamp
- Private skills: fully encrypted (metadata included)

### 4.2 Marketplace

- Public skill publishing with distribution key mechanism
- Search/browse by category, rating, usage
- Clone (read-only copy) and Fork (independent copy)

### 4.3 Claude Skill Interoperability

- **Import**: Accept Claude SKILL.md bundles (SKILL.md + scripts/ + references/ + assets/)
  - No steps.json → runs via Agent Track only
  - Over time, execution recordings build up steps.json automatically
- **Export**: Strip browser-specific fields (steps, executions, score)
  - Keep SKILL.md + resources → valid Claude Skill bundle
- **Marketplace format**: Skills distributed as zip bundles following Claude's packaging

### 4.4 Cross-device Migration

- ExportWrappedKey / ImportWrappedKey UI
- QR code or text-based key transfer
